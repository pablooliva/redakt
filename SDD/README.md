# SDD — Specification-Driven Development Artifacts

This directory contains artifacts generated by the **SDD (Specification-Driven Development)** workflow, an AI-assisted development process that drives features through structured phases: Research, Planning, Implementation, and Review.

## How It Works

Each feature goes through:

1. **Research** — Systematic codebase investigation, stakeholder analysis, edge case discovery
2. **Planning** — Specification creation with testable requirements, edge cases, and failure scenarios
3. **Implementation** — Code and tests written against the specification, with code reviews and critical reviews at each phase

Artifacts are generated by [Claude Code](https://claude.ai/code) using the SDD plugin and skill listed below.

## Tooling

- **[claude-skills](https://github.com/pablooliva/claude-skills)** — Claude Code skills, including the SDD Flow skill that coordinates the end-to-end workflow
- **[claude-plugins](https://github.com/pablooliva/claude-plugins)** — Claude Code plugins, including the SDD plugin that provides the individual phase commands (research, planning, implementation, review)

## Directory Structure

```
SDD/
├── research/        # RESEARCH-###-feature.md — codebase investigation and analysis
├── requirements/    # SPEC-###-feature.md — specifications with requirements, edge cases, failure scenarios
├── prompts/         # PROMPT-###-feature.md — implementation tracking documents
│   ├── implementation-complete/   # Final implementation summaries
│   └── context-management/        # Progress tracking across phases
└── reviews/         # Code reviews and critical reviews for each phase
```
